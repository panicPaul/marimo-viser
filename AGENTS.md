# Notebook Style Guide

When working on marimo notebooks in this repository:

- Keep code high level, clear, and easy to follow.
- Prefer functional and declarative code over implicit stateful logic.
- Use proper type annotations throughout, especially for reusable helpers.
- Choose explicit variable names; write names out instead of over-abbreviating.
- Keep cells small and focused.
- Prefer one meaningful helper, class, or concept per cell when practical.
- Preserve the notebook's existing structure unless there is a clear reason to change it.
- Keep related concepts grouped together so the notebook remains easy to read.
- Add concise docstrings to functions.
- Optimize for readability first, then performance, then cleverness.
- Use marimo check on the notebook
